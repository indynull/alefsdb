# alefsdb P6 + P7 — Production-oriented polish

**Goal:** Close research backlog (P6) and harden the single-node daemon into something operable (P7). Still **single-node, no multi-tenant ACLs, no replication** — “production-grade local store,” not a distributed product.

## P6 — Capability extensions

| ID | Deliverable | Exit criteria |
| --- | --- | --- |
| P6.1 | **Atomic multi-op transactions** | `Batch` RPC / `txn` CLI applies N ops under one WAL commit; all-or-nothing |
| P6.2 | **Secondary indexes** | Type + path indexes maintained on write; query uses them when predicates allow |
| P6.3 | **Persistent RPC client** | `Client` keeps a socket open; supports multi-call without reconnect |
| P6.4 | **Query tooling** | Optional `member` predicate; richer `get` display for structures (CLI) |

## P7 — Operability / production polish

| ID | Deliverable | Exit criteria |
| --- | --- | --- |
| P7.1 | **Data-dir flock** | Second `serve` fails fast if dir already locked |
| P7.2 | **Concurrent connections** | Accept loop spawns handlers; mutations still serialized by `DbHandle` mutex |
| P7.3 | **Graceful shutdown** | SIGINT/SIGTERM stops accept, drains, removes socket, releases lock |
| P7.4 | **Stats / health** | `Ping` + `Stats` RPC (keys, uptime, op counters) |
| P7.5 | **Structured logging** | `tracing` with env filter (`RUST_LOG`) |
| P7.6 | **Bench production path** | Persistent client + true multi-request-per-connection |
| P7.7 | **Docs** | README ops section: runbook, signals, locking, txn, bench |

## Non-goals (still)

- Multi-node replication / consensus
- Full SQL / Redis protocol
- Multi-user ACL model
- macFUSE first-class support
- Query-language **writes** (mutations stay RPC/CLI ops + batch)
